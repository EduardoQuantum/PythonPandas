1. Acceso a Visual Studio Code mediante Workspace

Para iniciar un proyecto de Ciencia de Datos de forma profesional, el primer paso es la correcta apertura del entorno desde la terminal del sistema. Esto garantiza que todas las rutas relativas de nuestros scripts de Python y archivos de datos funcionen correctamente.

En proyectos complejos, no basta con abrir una carpeta. Utilizamos archivos de Workspace para asegurar que Visual Studio Code mantenga la configuración de paneles, variables de entorno y extensiones específicas para el procesamiento de datos.

1.1 ¿Qué es un Workspace?

Es un archivo de configuración con la extensión ‘.code-workspace’. Es recomendable tener uno ajustado para cada proyecto. Al centralizar la configuración de Python, el Workspace asegura que el editor siempre apunte al intérprete y entorno virtual de nuestro interés, evitando conflictos entre proyectos.

Aqui abajo un detalle de la configuracion ‘.code-workspace’

    "settings": {
        // Interprete de Python apuntando al venv local de CursoPandas
        "python.defaultInterpreterPath": "${workspaceFolder}/venv/Scripts/python.exe",
        "code-runner.runInTerminal": true,
🆘 Mesa de Ayuda:

El archivo .code-workspace es un archivo de extensión JSON de configuración y es un poco “tiquismiquis” con las sangrias y los espacios. Una buena practica es pedirl a la IA que ajuste el archivo .code-workspace por nosotros.

Zona de Prompting: “Actúa como experto en Windows y Visual Studio Code. Genera el contenido para un archivo ProyectoPandas.code-workspace. Mi carpeta de proyecto es C:\CursoPandas y mi entorno virtual está en C:\CursoPandas\venv\. Asegúrate de que el python.defaultInterpreterPath apunte correctamente a Scripts/python.exe y que el terminal reconozca el venv automáticamente al abrir el espacio de trabajo. Devuélveme el código en formato JSON.”

🆘 Mesa de Ayuda: El Workspace no reconoce mi venv

Si después de usar el prompt y crear el archivo, VS Code no activa el entorno automáticamente:

  1. Cierra y abre: VS Code a veces necesita un “reinicio” para leer los cambios del workspace.
  2. Selección Manual: Presiona Ctrl + Shift + P, busca “Python: Select Interpreter” y elige el que está dentro de la carpeta venv.
  3. Rutas: Verifica que en el archivo no haya quedado ninguna barra invertida simple (\). En los archivos de configuración de VS Code, se deben usar barras inclinadas (/) o doble barra (\\).

1.2 Apertura desde la Terminal (CMD)

Lanzar el archivo ‘ProyectoPandas.code-workspace’ desde la terminal es el método más avanzado: carga configuraciones específicas y rutas predefinidas de un solo golpe.

Instrucciones: Abrir la terminal (Tecla Win + R, escribir ‘cmd’ y Enter).

# --- TERMINAL DE WINDOWS (CMD) ---

# 1. Navegar a la raíz del disco C:
cd \

# 2. Entrar a la carpeta del proyecto
cd C:\CursoPandas

# 3. Lanzar el Workspace de Visual Studio Code
code ProyectoPandas.code-workspace
Lo que estamos haciendo
  1. Navegación: Usamos ‘cd ' para ir a la base del disco y luego ’cd C:’ para entrar en nuestra carpeta de trabajo.
  2. Lanzamiento: El comando ‘code’ abre VS Code, y al pasarle el nombre del archivo ‘.code-workspace’, el editor se abre con todas nuestras preferencias ya cargadas.

Zona de Prompting: “Actúa como experto en Windows y Ciencia de Datos. Explícame el paso a paso para ingresar a mi proyecto en Visual Studio Code desde la terminal de Windows (CMD), considerando que mi carpeta de trabajo es C:y utilizo un archivo .code-workspace.”

2. Apertura de la base SAV

Para trabajar con datos provenientes de software estadístico, utilizaremos el formato ‘.sav’ (SPSS). En Python, la herramienta estándar es Pandas, pero para leer estos archivos necesita un motor especializado llamado ‘pyreadstat’.

2.1 Gestión del Entorno Virtual (venv)

Para asegurar que nuestro proyecto sea estable, instalaremos todas las librerías exclusivamente dentro de ’C:'. Este entorno es una copia aislada de Python que no afectará al resto del sistema.

Regla de Oro

Nunca ejecutes ‘pip install’ sin verificar que el entorno virtual está activo. Sabrás que está activo porque verás el prefijo (venv) al principio de la línea de comandos.

# --- GESTIÓN DE ENTORNO VIRTUAL ---

# 1. Asegurarse de estar en la carpeta del curso
cd C:\CursoPandas

# 2. Activar el entorno virtual (Sintaxis CMD)
venv\Scripts\activate

# 3. Actualizar el gestor de paquetes
python -m pip install --upgrade pip

# 4. Instalar dependencias necesarias
pip install pandas pyreadstat
Lo que estamos haciendo
  1. Activación: Al ejecutar el script ‘activate’, le decimos a la computadora que use el Python que está dentro de nuestra carpeta ‘venv’.
  2. Actualización: Ponemos al día ‘pip’ (el instalador) para evitar errores de compatibilidad.
  3. Instalación: Descargamos e instalamos ‘pandas’ (para tablas) y ‘pyreadstat’ (para archivos de SPSS).
¿Qué es pip?

pip (Pip Installs Packages) es el gestor de paquetes de Python. Nos permite descargar librerías adicionales desde el repositorio oficial (PyPI). Al usarlo dentro de nuestro ‘venv’, las librerías quedan disponibles únicamente para este proyecto.

3. Ingesta de Datos con Intérprete Vinculado

Al usar el archivo ‘.code-workspace’, VS Code detecta automáticamente nuestro entorno en ’C:'. Ahora podemos ejecutar nuestro script de carga.

# --- CARGA DE BASE EPH ---

import pandas as pd

# Definición de ruta con Raw String (r)
ruta = r'c:\CursoPandas\data\input\24Q4_INDIVIDUOS.sav'

# Carga de datos utilizando el motor instalado
df = pd.read_spss(ruta)

# Inspección inicial
print(df.head())
Lo que estamos haciendo
  1. Importación: Cargamos la librería bajo el apodo ‘pd’.
  2. Ruta Cruda: Usamos el prefijo ‘r’ en la ruta para que Python no se confunda con las barras ’' de Windows.
  3. Lectura: El comando ‘read_spss’ busca el archivo y lo transforma en un DataFrame llamado ‘df’.
  4. Verificación: ‘.head()’ nos muestra las primeras 5 filas para confirmar que todo se ve bien.

3.1 Análisis Técnico

  1. Importación (‘import pandas as pd’): Cargamos Pandas con el alias estándar ‘pd’.
  2. Rutas con “Raw Strings”: El prefijo ‘r’ antes de las comillas indica que es un texto “crudo”, evitando errores con las barras invertidas en Windows.
  3. Ingesta mediante motor externo: Utilizamos la librería ‘pyreadstat’ instalada en nuestro venv.
  4. Inspección de Integridad: ‘.head()’ nos devuelve las primeras 5 filas para confirmar la carga exitosa en la RAM.
Buenas Prácticas de Archivos

Es fundamental que los nombres de archivos no contengan espacios. Se recomienda usar guion bajo (’_’) y evitar caracteres especiales.

El Objeto DataFrame

Cuando asignamos la carga a ‘df’, creamos un DataFrame. Es una estructura bidimensional optimizada para procesar grandes volúmenes de información que superan las capacidades de una planilla de cálculo tradicional.
