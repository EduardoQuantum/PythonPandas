1. Acceso a VS Code mediante Workspace

Para iniciar un proyecto de Ciencia de Datos de forma profesional, el primer paso es la correcta apertura del entorno desde la terminal del sistema. Esto garantiza que todas las rutas relativas de nuestros scripts de Python y archivos de datos funcionen correctamente.

En proyectos complejos, no basta con abrir una carpeta. Utilizamos archivos de Workspace para asegurar que Visual Studio Code mantenga la configuración de paneles, variables de entorno y extensiones específicas para el procesamiento de datos.

1.1 ¿Qué es un Workspace?

Es un archivo de configuración con la extensión .code-workspace. Es recomendable tener uno ajustado para cada proyecto. Al centralizar la configuración de Python, el Workspace asegura que el editor siempre apunte al intérprete y entorno virtual de nuestro interés, evitando conflictos entre proyectos.

1.2 Apertura desde la Terminal (CMD)

Lanzar el archivo ProyectoPandas.code-workspace desde la terminal es el método más avanzado: carga configuraciones específicas y rutas predefinidas de un solo golpe.

Instrucciones: Abrir la terminal (Tecla Win + R, escribir ‘cmd’ y Enter).

# --- TERMINAL DE WINDOWS (CMD) ---
# 1. Navegar a la raíz del disco C:
cd \
# 2. Entrar a la carpeta del proyecto
cd C:\CursoPandas
# 3. Lanzar el Workspace de Visual Studio Code
code ProyectoPandas.code-workspace

  1. Apertura de la base SAV Para trabajar con datos provenientes de software estadístico, utilizaremos el formato .sav (SPSS). En Python, la herramienta estándar es Pandas, pero para leer estos archivos necesita un motor especializado.

2.1 Gestión del Entorno Virtual (venv) Para asegurar que nuestro proyecto sea estable, instalaremos todas las librerías exclusivamente dentro de C:. Este entorno es una copia aislada de Python que no afectará al resto del sistema.

Importante

Regla de Oro Nunca ejecutes pip install sin verificar que el entorno virtual está activo. Sabrás que está activo porque verás el prefijo (venv) al principio de la línea de comandos.

# --- GESTIÓN DE ENTORNO VIRTUAL ---

# 1. Asegurarse de estar en la carpeta del curso
cd C:\CursoPandas

# 2. Activar el entorno virtual (Sintaxis CMD)
venv\Scripts\activate

# 3. Actualizar el gestor de paquetes
python -m pip install --upgrade pip

# 4. Instalar dependencias necesarias
pip install pandas pyreadstat
Importante

¿Qué es pip? pip (Pip Installs Packages) es el gestor de paquetes de Python. Nos permite descargar librerías adicionales desde el repositorio oficial (PyPI). Al usarlo dentro de nuestro venv, las librerías quedan disponibles únicamente para este proyecto.

# --- USO DE PIP EN ENTORNO VIRTUAL ---

# 1. Usamos pip para traer la herramienta de lectura SAV
pip install pyreadstat

# 2. Verificamos qué librerías tenemos instaladas actualmente
pip list
  1. Ingesta de Datos con Intérprete Vinculado Al usar el archivo .code-workspace, VS Code detecta automáticamente nuestro entorno en C:. Ahora podemos ejecutar nuestro script de carga.
# --- CARGA DE BASE EPH ---

import pandas as pd

# Definición de ruta con Raw String (r)
ruta = r'c:\CursoPandas\data\input\24Q4_INDIVIDUOS.sav'

# Carga de datos utilizando el motor instalado
df = pd.read_spss(ruta)

# Inspección inicial
print(df.head())

3.1 Análisis del Script de Carga Importación (import pandas as pd): Cargamos Pandas con el alias estándar pd.

Rutas con “Raw Strings”: El prefijo r antes de las comillas indica que es un texto “crudo”, evitando errores con las barras invertidas en Windows.

Ingesta mediante motor externo: Utilizamos la librería pyreadstat instalada en nuestro venv.

Inspección de Integridad: .head() nos devuelve las primeras 5 filas para confirmar la carga exitosa en la RAM.

Importante

Buenas Prácticas de Archivos Es fundamental que los nombres de archivos no contengan espacios. Se recomienda usar guion bajo (_) y evitar caracteres especiales.

Nota

El Objeto DataFrame Cuando asignamos la carga a df, creamos un DataFrame. Es una estructura bidimensional optimizada para procesar grandes volúmenes de información.
