1. Cargamos la base SAV

Iniciamos nuestro script asegurando la carga de la librería y el acceso a los datos de la EPH.

**# --- CARGA DE BASE EPH ---**

import pandas as pd
# Definición de ruta con Raw String (r)
ruta = r'c:\CursoPandas\data\input\24Q4_INDIVIDUOS.sav'
# Carga de datos utilizando el motor instalado
df = pd.read_spss(ruta)
# Inspección inicial
print(df.head())
Comentarios: El uso de # vs ““”

Antes de avanzar, reparen en el uso de los comentarios. En Python, existen dos formas principales de documentar nuestro código y cada una tiene un propósito distinto:

  1. El numeral (#): Se utiliza para comentarios de una sola línea. Es ideal para explicaciones breves. Python ignora todo lo que escribas a la derecha del #.
    • Ejemplo: df.head() # Muestra las primeras 5 filas
  2. Las triples comillas (“““): Se conocen como docstrings o comentarios multilínea. Se usan para bloques de texto extensos o para desactivar temporalmente varias líneas de código.

Usaremos ambos todo el tiempo para comentar, especificar y organizar nuestro flujo de trabajo.

2. Comandos de Inspección: ¿Qué tenemos en la base?

Antes de operar, debemos “mirar” la base. El primer paso es conocer la naturaleza de nuestras columnas (variables) y el tipo de dato que almacenan.

**# --- INSPECCIÓN INICIAL ---**

# 1. ¿Cómo se llaman las columnas y qué tipo de datos son?
print(df.info()) 
# 2. ¿Qué aspecto tienen los datos? (Primeras 5 filas)
print(df.head())
# 3. ¿Cuántas filas y columnas tiene el archivo total?
print(df.shape)

2.1 Análisis de los Comandos de Inspección

Estos tres comandos son el “estetoscopio” del científico de datos para entender la estructura del DataFrame.

1. El Diagnóstico Estructural: .info()

print(df.info())

  • Sintaxis: Es un método. Lleva paréntesis al final porque realiza una acción de procesamiento.
  • Qué devuelve: El listado de columnas, valores no nulos y el Dtype (tipo de dato técnico).
    • float64 o int64: Números (Edad, Ingresos).
    • object o category: Texto o etiquetas.
  • Importancia: Nos asegura que las variables numéricas sean reconocidas como tales para poder hacer cálculos.

2. La Vista Previa: .head()

print(df.head())

  • Sintaxis: Método con paréntesis. Por defecto muestra 5 filas; df.head(10) mostraría diez.
  • Qué devuelve: Una tabla reducida para verificar visualmente que la carga fue exitosa.

3. La Dimensión del Objeto: .shape

print(df.shape)

  • Sintaxis: Es un atributo, por eso no lleva paréntesis. Representa una propiedad física del objeto.
  • Qué devuelve: Una “tupla” con dos números: (filas, columnas).
Diferencia Técnica: Método vs Atributo

Es un error común intentar usar df.shape(). Recordar:

  • Acción (procesar/calcular): Lleva paréntesis: .info(), .head().
  • Descripción física: No los lleva: .shape, .columns.

3. Profundización Técnica: Atributos vs. Métodos

En Python, un objeto como nuestro df tiene características (atributos) y habilidades (métodos).

3.1 Los Atributos (Descripciones)

Un atributo es una variable interna del objeto que guarda información sobre él. El objeto ya la conoce, por lo que no realiza cálculos, solo “la muestra”.

  • Sintaxis: objeto.atributo (Sin paréntesis).
  • Ejemplos: .shape, .columns, .index.

3.2 Los Métodos (Acciones)

Un método es una función que pertenece al objeto. Representa una acción que el objeto debe ejecutar. Los paréntesis disparan esa ejecución.

  • Sintaxis: objeto.metodo() (Con paréntesis).
  • Ejemplos: .head(), .info(), .describe().

4. Análisis de Variables Específicas

Nos enfocaremos en ‘CH06’ (Edad) y ‘CH04’ (Sexo).

4.1 La columna “Edad” (CH06)

**# --- EXPLORANDO EDAD ---**

# ¿Qué hay en edad? (Ver los valores únicos)
print(df['CH06'].unique())
# Medidas de tendencia central (Promedio, Mediana, Mínimo, Máximo)
print(df['CH06'].describe())

4.2 La columna “Sexo” (CH04)

**# --- EXPLORANDO SEXO ---**

# Frecuencias absolutas (Conteo: 1 = Varón, 2 = Mujer)
print(df['CH04'].value_counts())

5. Selección y Filtrado: Paso a Paso

Paso 1: Filtrar por Varones

Queremos ver la edad solo de los varones (‘CH04’ == 1.0).

**# --- FILTRO SIMPLE ---**

# Seleccionamos filas donde sexo es varón
varones = df.query('CH04 == 1.0')
# ¿Cuántos varones hay y cuál es su edad promedio?
print(f'Total varones: {len(varones)}')
print(f'Edad promedio varones: {varones["CH06"].mean():.2f}')

Paso 2: Filtro Combinado (Varones + Región)

Filtramos Varones de la Región GBA (‘REGION’ == 1.0).

**# --- FILTRO COMPUESTO ---**

# Filtramos por dos condiciones
varones_gba = df.query('CH04 == 1.0 and REGION == 1.0')
# Cuantificamos el resultado
print(f'Varones en GBA: {len(varones_gba)}')
print(f'Edad promedio en GBA: {varones_gba["CH06"].mean():.2f}')

6. Resumen de Herramientas Utilizadas

Comando Función
.info() Muestra tipos de datos e integridad.
.describe() Devuelve medidas de tendencia central.
.value_counts() Cuenta frecuencias por categoría.
.query() Filtra filas según condiciones lógicas.
Tip del Profesor

‘CH06’ (Edad) es numérica (podemos calcular promedios). ‘CH04’ (Sexo) es categórica (lo más útil es el conteo de frecuencias).
